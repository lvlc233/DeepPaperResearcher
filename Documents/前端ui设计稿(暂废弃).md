## 页面
### 首页
#### 组件
- 导航栏
- 搜索按钮
- 上传论文按钮
- 收藏按钮
- 登录按钮
- 项目图标
- 账号字符串
- 主页目录
- 主页主体
- 登录组件
- 注册组件
- 找回密码组件
#### 功能
1. 搜索功能: 点击搜索按钮跳转到 管理页面中的搜索模块中。
2. 上传论文: 点击上传论文按钮或将pdf拖入主页可以上传一篇pdf或一个文件夹或一个论文的url(指向pdf)，批量加载文件夹中的pdf内容。加载后引导用户去管理页面。
3. 游览功能: 游览主页的内容，可以用目录跳转，主页展示项目介绍。
4. 权限拦截，在进行搜索/上传/收藏等操作时，要求登录。
5. 去管理模块: 点击收藏跳转到管理模块中的收藏夹模块(初始默认)
6. 登录模块: 点击登录按钮跳转到登录模块，展示登录组件并提供相关登录功能
7. 注册模块: 点击注册按钮跳转到注册模块，展示注册组件并提供相关注册功能
8. 找回密码模块: 点击找回密码按钮跳转到找回密码模块，展示找回密码组件并提供相关找回密码功能
#### 描述
主页是用户进来的第一个地方，向用户提供引导功能
1. 导航栏部分: 导航栏位于页面最上面，在导航栏的左边,是项目图标和名称，右边是登录按钮，居中有3个按钮分别是搜索论文，上传论文和收藏。
2. 主体部分: 主体部分位于导航栏下面，可以滚动，并在页面左边悬浮主页的目录。目录可展开或收起。
登录注册和找回密码组件类似，都是左边静图片，右边标题＋输入框和其他组件跳转。
3. 登录部分: 点击登录在对应的页面展开登录组件，标题为登录，标题下方为邮箱+密码输入框，下方为文字描述的小号体注册和找回密码，分别居于两侧，再下方就是登录按钮。
4. 注册部分: 点击注册在对应的页面展开注册组件，标题为注册，标题下方为邮箱+密码+验证输入框，下方为文字描述的小号体去登录，居于右侧，再下方就是注册按钮。
5. 找回密码部分: 点击找回密码在对应的页面展开找回密码组件，标题为找回密码，标题下方为邮箱+新密码+验证码输入框，下方为文字描述的小号体去登录，居于右侧，再下方就是找回密码按钮。

- 搜索视图字段
- 管理页目录
- 管理页目录搜索按钮
- 管理页目录收藏栏组件
- 管理页目录收藏夹组件
- 收藏夹新增组件
- 收藏夹操作组件
- 管理页收藏夹视图组件
- AIapi设置组件。
#### 功能
0. 权限拦截，进行操作时确保登录。
1. 搜索功能: 在搜索栏中输入关键词，设置筛选条件和搜索设置(AI辅助)后点击搜索按钮，展示搜索结果视图组件。
2. 视图切换: 用户可以在搜索的设置中选择要展示的字段，之后页面就只渲染相关字段。也可以在设置中设置分页规则和AI搜索长度限制。
3. 筛选功能: 点击预定好的筛选条件，页面会根据条件筛选出符合条件的论文。
4. AI辅助: 在搜索栏启动搜索AI辅助功能后，AI搜索相关内容并同样渲染搜索结果视图。
5. 收藏功能: 在非收藏夹收藏的论文处点击收藏按钮，将论文添加到指定的收藏夹中。或对于已经收藏的论文可以点击取消收藏。
6. 上传论文: 点击上传论文或将pdf拖入收藏夹页面中可以上传一篇pdf或一个文件夹或一个论文的url(指向pdf)，批量加载文件夹中的pdf内容。之后加载论文到当前收藏夹中。
7. 收藏夹功能: 用户可以在管理页面中目录中管理文件夹和切换当前组件为收藏夹页面。

#### 描述
管理页面保存着用户的收藏夹和上传的论文，并同时提供外部搜索功能。
1. 导航栏部分: 导航栏位于页面最上面，在未登录的情况是登录按钮，同上，登录后右侧显示邮箱字符串，点击后可以弹出AI的配置组件用户配置API.供应商之类的。
2. 目录部分: 管理页面的目录不悬浮，而是在整个页面的左侧部分，具有展开和收起的功能。在目录中分为两个模块，最上方是搜索按钮，点击将主体部分切换为搜索时的组件页面。第二个部分是收藏夹模块，展示用户的收藏夹。收藏夹中存在一个默认收藏夹，用户不可以删除该收藏夹，所有直接上传的论文都会放在默认收藏夹中。用户可以点击收藏夹栏中的+创建一个新的收藏夹并命名。收藏夹显示名字和当前收藏夹论文数量。点击某个收藏夹后面的操作符可以展开，对收藏夹重命名和删除收藏夹(论文一并移除提醒)。
3. 主体部分: 主体部分主要由搜索栏和搜索结果组成。并对收藏夹/搜索/AI搜索具有区分。
    - 通用部分: 在该组件的中上方有一个搜索栏,搜索栏右边是设置按钮。在搜索栏内容有一个AI辅助的开关。并且在搜索栏下方有若干条件的多选，一般和论文相关的数据有关。点击设置可以配置分页规则、展示字段和搜索来源和AI搜索的参数，例如搜索长度限制等。可以配置字段至少包括一个标题。当有数据时候，根据搜索的结果和当前页面的配置进行表格的加载。在点击表格中标题时候，跳转到阅读详情页面。
    - 限制收藏夹时候: 当在收藏夹页面时，搜索结果只展示该收藏夹中的论文。默认不搜索时候，全量的展示该收藏夹中的论文。并在页面的右上角显示当前的收藏夹名。点击表格中的最右侧按钮，可以移除该论文在收藏夹中。在该页面中，可以店家搜索栏下方的上传论文，来直接将一篇论文添加到该收藏夹中。(参考字段: 标题,注释,作者,来源,期刊,发表日期,摘要,标签,添加时间)
    - 搜索时: 当在搜索页面，那么此时就会根据配置的搜索来源进行搜索(默认包括所有的收藏夹,可以通过条件进行设置)，并将搜索的结果在页面中展示。不可上传论文。右上角不显示任何东西。可以对搜索到的结果进行收藏，并指定收藏的收藏夹在哪?可以直接创建新的收藏夹。或移除该收藏(如果搜索到的结果在收藏夹中的话)(参考字段: 标题,注释,作者,来源,期刊,发表日期,摘要,标签,添加时间)
    - AI辅助时: AI辅助不是一个单独的页面，而是对该页面有修正效果，会新增一个AI推荐原因和分数排名。AI会持续的进行搜索，并传递新的数据，Ai会发送事件，直到结束前，搜索栏的左侧会显示转圈，直到发送搜索结束事件，每次事件就是一次推荐的结果。用户可以随时的进行暂停。并保留当前AI搜索的结果。
    - 搜索在发起前都会保留上一次搜索的结果如果在相同的页面中的话。


### 阅读
#### 组件
- 导航栏
- 收藏/取消按钮
- 搜索栏
- 视图栏
- 新创视图按钮
- 目录
- 论文阅读器
- 阅读辅助组件
- 阅读辅助组件顶部栏
- AI聊天组件
- 笔记组件
- 脑图组件
- 报告组件
- 设置组件
- 聊天输入框组件
- 聊天输入框顶部指令按钮
- 对话组件
#### 功能
1. 搜索功能: 在搜索栏中输入关键词，在该论文内进行搜索并导航到该对应位置。
2. 目录功能: 可以通过点击目录快速的切换到论文的不同区域。
3. 收藏功能: 可以在该页面选择收藏或取消收藏该论文。
4. 阅读功能: 可以在主体部分进行阅读和操作
5. 视图管理功能: 解释视图概念:视图相当于一个玻璃，用户对论文的所有标记注释等操作，都将保存在视图层，而不直接对论文进行修改。通过叠加移除等操作，让用户可以在方面的管理自己对论文的操作。用户可以创建查看叠加修改视图。
6. AI聊天: 在进入论文时，会自动的加载论文到AI系统中并生成摘要和脑图。用户可以在导读模块和AI进行聊天，用户可以上传自己的视图或笔记给AI或指定论文中的某一段让AI去阅读和对话。
7. 深度研究: 这是聊天组件的一个子功能,开启该功能后，AI会向用户明确研究内容，并开始研究后无阻塞的进行研究，研究期间用户可以继续正常对话，并在报告中生成研究任务条目，并生成报告。并在报告组件查看研究结果。
8. 笔记功能: 用户可以进行笔记的书写。



#### 描述
阅读页是项目的主要部分,用于辅助用户进行阅读
1. 在页面的最上方是导航栏,居中的是文章内搜索，在搜索的左边是收藏或取消按钮。在搜索的右边是视图管理按钮，可以打开选择新建,删除，重命名，选中(叠加)等操作。
2. 页面左侧是导航栏，可以收起和展开，点击具体的标签可以跳转
3. 在主体部分，提供大量的工具，并标清当前操作视图和叠加的视图集，可以切换当前的视图进行对应不同视图的操作或AI相关操作(例如选取)。点击翻译在主体部分分为左右两个部分进行翻译对照(可分配占比)
4. 在页面右侧是阅读辅助组件，该组件有两部分组成，上方的辅助组件的导航栏,包括导读,笔记，脑图,报告,设置。剩下的部分不同组件对应的主体
    - 导读: 起始就是导读位置,在导读加载完成后，会生成一段摘要,之后就可以进行对话了，主体的最下方是输入框，用于输入对话信息，对话框的顶部区域是一个功能栏，表示一些功能,当前仅有深度思考功能。正常对话时候，用户的消息在右侧Ai的消息在左侧，AI的消息是流式输出的，并附加消息发送时间，用户可以从任意的地方重新发送消息。参考kimi的编辑对话功能，对话部分就是常见的AI对话。点击深度思考时，前面同样正常交流，在AI确认了研究任务后会生成一份研究方向并等待你的确认，在确认完成后会在后台挂起一个任务，可以在报告中查看。
    - 笔记: 用户可以在笔记模块进行记录,基于md
    - 脑图: AI生成的脑图，可以让他重新生成，但仅读。
    - 报告: 报告处存放的时Ai的研究任务，有4个状态,等待，开始,完成和异常。最终的结果也会是一份md报告
    - 设置: 这里可以设置深度研究时的模型配置(默认用户和正常聊天同级)，生成脑图和对话的时候的模型配置，例如一些API或提示词。
